win平台使用emacs

发表于 2019-05-18 更新于 2024-06-23
最早在archlinux上使用emacs，然而日常工作环境要求用win，所以针对win平台做了很多配置优化和功能增强，简单的记录当前我日常的使用情况。

我都用emacs在win上干什么

打开应用，或快速查找文件，打开文件夹等

利用 w32 api 可以直接调用系统的相关功能块，完成一些自己需要的功能，比如当我需要在当前应用前端打开某个应用时不需要先win+d显示桌面,再找图标双击.
git-bash 是我用的最多的程序，打开方式是 emacs-> f2+o -> bash,相较于在文件夹中右键 git-bash here 快了很多倍

写一点小脚本工具

当我经常要用到某种功能的时候就自己写个小脚本保存起来，txt,notepad++也能写，不过emacs中我集成了 fast-run，写完就可以马上运行验证，不用开一个独立的bash. 最关键的是我自己拓展了emacs的 eshell，可以让我的emacs像 intellij ide一样有bottom terminal风格，也可以快速执行。(eshell 在win上表现很无力，所以我一般还是使用单独的 git-bash)

org-mode 做gtd

这个没什么好说的，大部分使用emacs人都在用的功能.

其他一些小辅助功能

calc 进制转换，绑定了emacs里面 f2-f4关机,spc+f 快速查找桌面文件 alt+0+d 删除文件,使用python调用平台提醒功能定时提醒,这些都是一些偶尔想到的拓展,探索中总会发现新的惊喜。
